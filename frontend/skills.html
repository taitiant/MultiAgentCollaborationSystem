技能策略

Skill 负责增强智能体的方法论、约束和 capability 使用策略。

能力执行

技能列表

点击左侧 skill，在右侧编辑它的方法论与偏好能力。

设计原则

Skill 影响“怎么做”，Capability 影响“能怎么做”。

Skill 不是权限 Agent 可以直接调用 capability，skill 主要用于注入方法论、约束和调用策略。
偏好 vs 必需 `preferred_capabilities` 用于建议搭配；`required_capabilities` 用于声明没有这些能力时 skill 无法完整落地。
恢复默认 可把当前 skill 重置到系统默认配置，便于收敛阶段策略。
